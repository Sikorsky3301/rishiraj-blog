## Summary

The video features Garrett Galow from WorkOS, who discusses the challenges of answering business questions that require SQL. He explains that the traditional process involves a loop of explaining the question, waiting for an engineer, getting an answer, and then realizing that the query needs further modification. To break this loop, Galow built Studio, an internal workspace that allows anyone to ask questions against Snowflake, Linear, and Notion in natural language and receive answers or reusable widgets without filing a request.

The key innovation in Studio is the use of widgets that are generated by a large language model (LLM) as declarative JavaScript code. This code calls the underlying data sources directly, making every subsequent run deterministic and cheap. The widgets are reliable enough to be handed over to a support team, thanks to three key features: preflight sequencing, layering rules, and query validation. Preflight sequencing injects schema context only when a tool is invoked, keeping the context window clean. The layering rule explicitly tells the model to distrust its own knowledge about WorkOS and instead go to primary sources.

The query validation feature is particularly important, as it runs every generated Snowflake query before hardcoding it into a widget. This catches the "valid SQL that returns zero rows" failure mode, which can be difficult to detect. By using these features, Studio is able to provide accurate and reliable answers to business questions, without requiring extensive technical expertise. This has the potential to greatly improve the efficiency and effectiveness of business decision-making, by allowing anyone to access and analyze data without having to rely on engineers.

The use of natural language processing and LLMs in Studio also highlights the potential for AI to augment human capabilities in data analysis. By generating reusable widgets and providing answers in natural language, Studio is able to bridge the gap between technical and non-technical stakeholders. This can help to facilitate collaboration and communication across teams, and can ultimately lead to better business outcomes. Overall, the technical insights and takeaways from Galow's talk provide a compelling vision for the future of data analysis and business decision-making.

The implications of Studio's approach are significant, as it has the potential to democratize access to data and analytics across organizations. By providing a platform that allows anyone to ask questions and receive answers in natural language, Studio can help to break down silos and facilitate collaboration between teams. This can lead to faster and more informed decision-making, and can ultimately drive business success. As the use of AI and LLMs in data analysis continues to evolve, it will be interesting to see how Studio's approach is adopted and adapted by other organizations.

## Description

Every business question that needs SQL follows the same loop: explain the question, wait for an engineer, get an answer, realize it needs one more join, share a one-off in Slack, repeat. Garrett Galow from WorkOS built Studio to break that loop — an internal workspace where anyone can ask questions against Snowflake, Linear, and Notion in natural language and get answers or reusable widgets without filing a request.

The widgets are the interesting part: the LLM writes them once as declarative JavaScript that calls the underlying data sources directly, so every subsequent run is deterministic and cheap. Three things made it reliable enough to hand to a support team. Preflight sequencing that injects schema context only at the moment a tool is invoked, not upfront, keeping the context window clean. A layering rule that explicitly tells the model to distrust its own knowledge about WorkOS and go to primary sources. And query validation that runs every generated Snowflake query before hardcoding it into a widget, catching the valid SQL that returns zero rows failure mode.
