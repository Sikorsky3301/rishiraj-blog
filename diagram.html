02 · Local Machine
⚡
fetch.ps1
  • PowerShell script — run manually
  • Loads .env.local (secrets)
  • Installs Python dependencies
  • Calls fetch_videos.py
⚙
scripts/fetch_videos.py
  • feedparser — parse RSS XML
  • Processes latest 5 videos per channel
  • Checks Supabase for duplicates
  • Extracts: title, date, description, video_id
  • Inserts new rows into Supabase
🔒
.env.local
  • SUPABASE_URL
  • SUPABASE_SERVICE_KEY secret
  • YOUTUBE_RSS_URLS
  • Gitignored — never pushed
REST API / HTTPS (Service Key)
03 · Database — Supabase (PostgreSQL)
⬡
posts table
id uuid PRIMARY KEY
video_id text UNIQUE
slug text UNIQUE
title text
date date
channel text
tech_stack text
summary text
description text
created_at timestamptz
🛡
Row Level Security
  • RLS enabled on posts table
  • Public READ anon key
  • INSERT/DELETE service key
  • Frontend can only read
  • Script can write
⇄
REST API
  • Auto-generated by Supabase
  • Endpoint: /rest/v1/posts
  • Supports filtering, ordering, pagination
  • Used by both script and frontend
REST API / HTTPS (Anon Key)

05 · Deployment
⎈
GitHub
  • Sikorsky3301/rishiraj-blog
  • Source of truth for all code
  • Every push triggers Vercel deploy
▲
Vercel
  • Detects Next.js via package.json
  • Runs: npm run build
  • Serves on vercel.app domain
  • Env vars: NEXT_PUBLIC_* keys
↻
ISR (Incremental Static Regen)
  • Pages pre-rendered at build time
  • Revalidated every 3600 seconds
  • New Supabase rows appear automatically
  • No redeploy needed for new posts
